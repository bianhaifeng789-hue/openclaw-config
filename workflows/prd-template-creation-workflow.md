# PRD 模板创建与优化工作流

适用场景：需要为团队创建一份可长期复用的固定 PRD 模板。

---

## 工作流程

### 1. 需求确认
- 确认用户需要：一份完整的、可复用的 PRD 模板
- 确认必须包含：流程图
- 确认使用场景：飞书文档 / 团队协作

### 2. 本地模板创建
- 参考现有模板：`PRD-template-advanced.md`（如果有）
- 创建完整版本：`PRD-template-complete-with-flowcharts.md`
  - 19 章完整结构
  - 流程图章节必填
  - 功能规格模板可复制

### 3. 优化为固定模板
- 明确规则：
  - 默认不删章节，只填内容
  - 没有内容的章节写"本期不涉及"
  - 流程图至少保留主流程图 + 异常流程图
- 创建飞书优化版：`PRD-template-feishu-fixed.md`
  - 标题层级适合飞书
  - 表格数量适中
  - 可直接复制

### 4. 飞书文档创建
- 使用 `feishu_doc` 工具：
  - `action: create` 创建文档
  - `action: write` 写入完整内容
  - `action: read` 验证写入结果
- 文档标题：`PRD 固定模板 - 团队复用版`
- 给用户添加编辑权限

### 5. 内容补全
- 追加使用指南：
  - 第 20 章：各章节填写要点
  - 第 21 章：评审 Checklist
  - 第 22 章：快速填写版（最小可接受）
  - 第 23 章：不同项目类型适配
  - 第 24 章：协作流程说明
  - 第 25 章：版本迭代建议
  - 第 26 章：文档质量检查
  - 第 27 章：模板维护建议

### 6. Git 提交
- 每次重要改动都提交 git
- 提交信息示例：
  - `Add complete PRD template with flowcharts`
  - `Refine PRD template into fixed reusable structure`
  - `Add Feishu-optimized fixed PRD template`

### 7. 交付确认
- 提供飞书文档链接
- 说明模板特点和使用方法
- 确认用户可以长期复用

## 工具使用要点

### feishu_doc 工具
- `action: create` - 创建文档，返回 document_id 和 url
- `action: write` - 写入完整内容（适合初次写入）
- `action: append` - 追加内容（适合补全）
- `action: read` - 读取验证内容完整性
- `action: list_blocks` - 查看文档结构

### 注意事项
- 飞书 API 有延迟，写入后建议等几秒再验证
- 大内容建议分批写入或直接用 `write` 写完整内容
- 表格、代码块在飞书中会自动转换
