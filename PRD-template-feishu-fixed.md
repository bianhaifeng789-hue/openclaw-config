# PRD-template-feishu-fixed.md

适合直接复制到飞书文档中的固定 PRD 模板。

特点：
- 标题层级清晰，适合飞书文档目录
- 表格数量适中，不会太难维护
- 流程图章节固定保留
- 可直接复制整篇使用
- 即使是轻量需求，也能保持统一结构

使用规则：
- 默认不要删章节，只填内容
- 没有内容的章节写“本期不涉及”或“无”
- 流程图章节至少保留主流程图和异常流程图
- 详细功能部分可按模块复制多份

## 8. 核心流程图

> 本章节固定保留，不能删除。

### 8.1 主流程图
```mermaid
flowchart TD
    A[用户进入页面] --> B{是否满足前置条件}
    B -->|否| C[提示并引导补齐条件]
    B -->|是| D[开始主任务]
    D --> E[系统处理中]
    E --> F{处理结果}
    F -->|成功| G[展示成功结果]
    F -->|失败| H[展示失败原因]
    G --> I[推荐下一步动作]
    H --> J[支持重试/反馈/返回]
```

### 8.2 关键分支流程图
```mermaid
flowchart TD
    A[进入核心流程] --> B{用户类型}
    B -->|新用户| C[展示引导说明]
    B -->|老用户| D[直接进入功能]
    C --> E[开始任务]
    D --> E
    E --> F{是否命中限制条件}
    F -->|是| G[弹出限制说明]
    F -->|否| H[继续执行]
```

### 8.3 异常流程图
```mermaid
flowchart TD
    A[用户发起操作] --> B{网络是否正常}
    B -->|否| C[提示网络异常]
    C --> D[重试]
    B -->|是| E{服务是否返回异常}
    E -->|是| F[展示错误码/兜底文案]
    F --> G[记录日志与埋点]
    E -->|否| H[继续主流程]
```

### 8.4 流程图填写要求
- 必须包含入口、动作、判断、结果、退出路径
- 至少覆盖成功、失败、不满足条件三种情况
- 超过 12 个节点时建议拆图
- 特殊模式差异必须单独标注
