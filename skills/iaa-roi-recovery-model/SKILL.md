# IAA 投放分析与回收模型

## 一、先说结论：你这个场景的核心

你现在说的两个产品线：
- **文件修复**：首日回收 85%
- **清理**：首日回收 65%

这说明你们本质上是：
- **超短回收模型**
- **极度依赖D0/D1变现**
- **留存差，后链路价值很薄**

所以模型重点不是传统 D30/D90 LTV，而是：

```text
D0 ROI / D1 ROI / D3 ROI / D7 ROI
+ 后续残值（tail value）
```

也就是说，你们最重要的是：
1. 首日能回多少
2. 次日还能补多少
3. 第3-7天尾部还能补多少
4. 由此反推：CPI最多能买到多少

---

## 二、适合你们的 LTV 模型结构

### 完整模型

```text
LTV_total = LTV_D0 + LTV_D1 + LTV_D3_tail + LTV_D7_tail + LTV_D30_tail
```

但对于低留存工具类/文件类产品，更实用的写法是：

```text
LTV_total ≈ D0价值 + D1残值 + D3残值 + D7尾值
```

其中：

```text
D0 ROI = D0 Revenue / Spend
D1 ROI = (D0 + D1 Revenue) / Spend
D3 ROI = (D0~D3 Revenue) / Spend
D7 ROI = (D0~D7 Revenue) / Spend
```

如果：

```text
D0 ROI = 85%
D7 ROI = 100%+
```

说明这类产品可投。

如果：

```text
D0 ROI < 65%
且 D3/D7 提升很小
```

基本就回不来。

## 三、每日ROI模型

### 公式

```text
Day N ROI = 累计收入(0~N) / 投放成本
```

例如：

- CPI = $1.00
- D0收入 = $0.85
- D1新增收入 = $0.08
- D2新增收入 = $0.03
- D3新增收入 = $0.02
- D4-D7新增收入 = $0.04

则：

```text
D0 ROI = 0.85 / 1.00 = 85%
D1 ROI = (0.85 + 0.08) / 1.00 = 93%
D2 ROI = 96%
D3 ROI = 98%
D7 ROI = 102%
```

这就是一个“首日接近回本，7天刚过线”的模型。

---

## 四、低留存产品的简化LTV模型

### 推荐拆解

```text
LTV = 首日价值 + 次日残值 + 3-7天尾值 + 8-30天忽略或按极低残值估算
```

### 可执行版本

```text
LTV_total = D0_ARPU + D1_ARPU + Tail_ARPU
```

其中：

```text
Tail_ARPU = (D3~D7累计收入) + (D8~D30极低尾部)
```

对你这类产品，通常：

```text
D0占总LTV：65% ~ 85%
D1占总LTV：10% ~ 20%
D3-D7占总LTV：5% ~ 15%
D8+占总LTV：0% ~ 5%
```

所以你们真正该看的，不是传统游戏那种长期LTV，而是：

```text
LTV_total ≈ D0 / 0.75
```

举例：

如果 D0 = $0.85，占总LTV的80%

```text
LTV_total = 0.85 / 0.8 = 1.0625
```

如果 CPI = $1.00，则：

```text
总ROI = 106.25%
```

可投。

如果 D0 = $0.65，占总LTV的80%

```text
LTV_total = 0.65 / 0.8 = 0.8125
```

那不管怎么补，都难回本。

## 五、你当前两个品类的判断模型

### 1）文件修复（首日85%）

```text
假设：
D0 = 85%
D1补 8%
D2-D7补 10%
总ROI ≈ 103%
```

结论：

```text
✅ 可投
✅ 属于“快回收”模型
✅ 可接受更激进的出价
```

### 2）清理（首日65%）

```text
假设：
D0 = 65%
D1补 7%
D2-D7补 8%
总ROI ≈ 80%
```

结论：

```text
❌ 通常回不来
❌ 除非CPI继续下降
❌ 或显著提高首日变现
```

这就是你说“低于这个都回不来本”的本质原因。

## 六、反推可接受 CPI

### 核心公式

```text
Max CPI = 预估LTV / 目标ROI要求
```

如果目标是保本：

```text
Max CPI = 预估LTV
```

如果目标是10%利润：

```text
Max CPI = 预估LTV / 1.1
```

### 用首日回收反推

如果你知道：

```text
D0 Revenue per install = $0.85
D0占总LTV的80%
```

则：

```text
LTV ≈ 0.85 / 0.8 = 1.0625
```

若要求：

```text
最终ROI ≥ 105%
```

则：

```text
Max CPI = 1.0625 / 1.05 ≈ 1.01
```

也就是说，**买量单价超过 $1.01 就危险**。

## 七、投放到底该用什么模式

你说的“roast”我判断你大概率是指：
- **ROAS模式**
- 或者平台里的 **tROAS / value optimization**
- 也可能是成本模式 vs 价值模式的选择

### 对你这类产品的建议

#### 结论先说

```text
低留存、超短回收产品：
优先用 成本控制 / D0价值优化 / 短周期价值回传
不要一开始就上长周期 tROAS
```

### 为什么

因为你们：
- 留存差
- 长期价值弱
- 大部分价值在 D0/D1

如果直接用传统 tROAS：
- 平台会倾向找“长期高价值用户”
- 但你没有足够长期事件喂给它
- 反而可能导致量级起不来，学习困难

## 八、不同平台投放模式建议

### Google Ads（UAC）

#### 早期

```text
优先：tCPI / Max Install Volume
不要直接上 tROAS
```

原因：
- 新产品事件少
- D7/D30数据不稳定
- tROAS学不出来

#### 中期

如果你能稳定回传 D0/D1 value：

```text
切到：value optimization / tROAS（短窗）
但目标要保守
```

建议：

```text
D0 ROI 85% 的文件修复：
- 前期：tCPI 跑量
- 稳定后：D0 value + 保守 tROAS

D0 ROI 65% 的清理：
- 先别上 tROAS
- 先压 CPI
- 先做创意和变现提升
```

### Facebook / Meta Ads

#### 早期

```text
优先：App Install / AEO（如果事件足够）
如果没有强事件，先走最低成本
```

#### 中期

如果你能稳定回传首日广告价值：

```text
用 Value Optimization / Minimum ROAS
但只适合文件修复这类 D0 高回收产品
```

不建议：

```text
清理类 65% D0 回收产品直接上 Value Optimization
因为学习会很差，量也容易塌
```

### TikTok Ads

```text
更适合先跑成本模式 / 最低成本
用素材强推首日高价值用户
先看 D0 / D1 再决定是否价值优化
```

## 九、到底设置多少

### Google / Meta 的目标设置思路

#### 文件修复（D0 85%）

这是可以试价值优化的品类。

##### 目标建议

```text
D0目标ROI：80%~85%
D1目标ROI：90%~95%
D7目标ROI：100%~105%
```

##### 出价建议

如果当前保本 CPI = $1.00

```text
测试出价：
- 保守：$0.80 ~ $0.85
- 正常：$0.85 ~ $0.95
- 激进：$0.95 ~ $1.00
```

##### tROAS/VO 设置

```text
不要一上来设 100%+
建议先从 70%~80% 起
稳定后拉到 85%~90%
```

因为平台目标设太高，量会掉得很厉害。

#### 清理（D0 65%）

##### 目标建议

```text
D0目标ROI：65% 只是生死线，不是优秀线
真实可投至少要往 75%+ 做
```

##### 出价建议

```text
如果当前总LTV只支撑 $0.75 CPI
那你买量目标CPI最好压到：
- 保守：$0.55 ~ $0.60
- 正常：$0.60 ~ $0.68
- 上限：$0.70 ~ $0.75
```

##### 模式建议

```text
优先最低成本 / tCPI
不要上价值优化
先做两件事：
1. 拉高D0变现
2. 压低CPI
```

## 十二、实操建议

### 文件修复

```text
投放模式：
- 先 tCPI / lowest cost
- 跑稳后切 D0 value optimization
- ROI目标先设低一点（70%-80%）
- 稳后再提

放量条件：
- D0 ≥ 85%
- D3 ≥ 95%
- D7 ≥ 100%
```

### 清理

```text
投放模式：
- 只用成本模式
- 严控CPI
- 不做激进放量

优化重点：
1. 提升首日广告价值
2. 强化激励视频触发率
3. 提升首日展示次数
4. 压低获客成本
```

文件修复 85% 首日回收：可做价值优化，小心放量。
