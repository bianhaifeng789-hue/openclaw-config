## 核心发现

### 1. 焦虑检测机制 ⭐⭐⭐
**创新点**：模型会"提前收工"（context anxiety），compaction不够必须reset

**实现细节**：
- 9个焦虑信号正则模式（"let me wrap up", "running low on context"等）
- Reset阈值：150k tokens
- Checkpoint写入：progress.md持久化

**对比结果**：OpenClaw 已实现焦虑检测（impl/bin/anxiety-detector.js），但**缺失 progress.md 持久化**。

### 2. 三级门控机制 ⭐⭐⭐
**创新点**：PreExitVerificationMiddleware 三级退出门控

**实现细节**：
- Gate 1: 未工作 → 强制开始（最多3次尝试）
- Gate 2: 已工作 → 强制验证（自动检查 workspace outputs）
- Gate 3: 已验证 → 允许退出

**自动检查项**：
- 空文件检测（stat.size == 0）
- TODO/NotImplementedError 搜索（grep -rl）
- Skeleton 文件检测

**对比结果**：OpenClaw 已实现三级门控（impl/bin/pre-exit-gate.js），**已增强自动检查功能**。

### 3. Smart Truncate Output ⭐⭐
**创新点**：截断输出时保留关键信息，避免丢失错误线索

**实现细节**：
- stderr 保留 40% 预算
- stdout head/tail 各 40%
- 中间提取 error/fail/assert/exception 行

**对比结果**：OpenClaw 已实现（impl/bin/smart-truncate.js）。

### 4. Task Tracking (_todo.md) ⭐⭐⭐
**创新点**：ForgeCode's todo_write enforcement，TB2 提升 38% → 66%

**实现细节**：
- 4次工具调用后强制创建 _todo.md
- 12次调用检查更新提醒
- 持久化到磁盘（抵抗 context reset）

**对比结果**：OpenClaw 已实现（impl/bin/task-tracking.js）。

### 5. Skeleton Detection ⭐⭐
**创新点**：检测骨架文件，强制填充而非创建新文件

**实现细节**：
- 扫描 TODO/NotImplementedError/FIXME 标记
- 支持多语言（py/c/cpp/java/rs/go）
- 迭代2-3触发

**对比结果**：OpenClaw 已实现（impl/bin/skeleton-detector.js）。

### 6. Error Guidance ⭐⭐
**创新点**：预置12个错误模式 + 修复建议

### Compaction 算法
- evaluator: 50% retention
- builder: 20% retention
- default: 30% retention

### Reset 算法
- 检测焦虑信号（≥2个匹配）
- 或 tokens > RESET_THRESHOLD（150k）
- 写入 checkpoint → 清空 messages → 注入 checkpoint + git diff
