# 竞品分析技能

当用户提到 **竞品分析、对标产品、拆解竞品、竞品研究、研究别人怎么做** 时，使用本技能。

---

## 1. 适用场景
- 新产品立项前竞品研究
- 已有产品优化时对标分析
- 功能设计前参考竞品
- 商业化 / 增长 / 体验拆解

## 2. 常见触发句
- “帮我分析这个竞品”
- “拆一下这个 App”
- “研究下别人怎么做的”
- “做个对标分析”
- “看看这个产品的优缺点”

## 3. 输入
支持输入：
- 产品名称
- App 链接 / 官网链接
- Google Play 链接
- APK / 截图 / 用户体验记录

## 4. 输入不足时
如果缺少核心材料，至少先补一个：
- 产品名称
- 下载链接 / 官网链接
- 截图 / 录屏
- 重点分析方向（功能 / 商业化 / 增长 / 技术）

若仍不足，应先输出“轻量竞品分析框架 + 待补信息”，不要假装完成深度分析。

## 5. 分析目标
至少回答：
- 竞品是谁，定位是什么
- 核心用户是谁
- 核心功能和工作流是什么
- 商业化和增长手段是什么
- 哪些地方值得学
- 哪些地方不建议照搬

## 6. 标准分析维度
- 产品定位
- 目标用户
- 核心场景
- 功能结构
- 页面与交互
- 商业化
- 增长策略
- 技术实现线索
- 优势 / 劣势
- 对我方启发

## 7. 推荐输出结构
```markdown
# 竞品分析
## 1. 基础信息
## 2. 产品定位
## 3. 用户与场景
## 4. 功能架构
## 5. 页面与体验
## 6. 商业化
## 7. 增长策略
## 8. 技术线索
## 9. 优势与短板
## 10. 对我方启发
```

## 8. 最低输出要求
至少必须给出：
- 竞品定位判断
- 3 个核心功能 / 机制
- 1 个商业化结论
- 1 个增长结论
- 3 条对我方启发

## 9. 输出原则
- 不只罗列功能，要归纳策略
- 不只说表面体验，要判断背后目标
- 要形成“借鉴 / 不借鉴”结论
- 能与 PRD / Gap 分析衔接

## 10. 联动技能
- 单竞品深度分析：由 `google-play-to-prd` 主流程完成后，需要多竞品横向对比时调用本 skill
- 已有逆向数据时：结合 `apk-reverse-analysis` 的输出做技术维度对标
- 要落到版本决策时：输出传给 `gap-analysis` → `prd-generator`
- 本 skill 是分析层，不负责 APK 获取和逆向执行
