## 1. 执行摘要

这是一个典型的“文件恢复 / 垃圾清理 / 相册整理”混合型工具 App。它在 Google Play 上以“误删照片/视频/文件恢复”为主卖点，但从商店文案、开发者官网和用户评论看，真实体验更像是：

1. 扫描当前存储与缓存中的可见文件，
2. 用“恢复”叙事包装结果，
3. 通过高频广告、通知和弱承诺获取流量变现。

从产品经理视角，它更值得被当作“流量型工具竞品”而不是“真正的数据恢复技术产品”来拆解。可借鉴的是它的需求包装、低门槛 onboarding 和多场景切入，不建议照搬的是其误导性价值承诺、广告打断和低信任体验。

## 4. 竞品拆解

## 4.1 核心定位

**一句话定位**

帮助用户“找回”误删照片、视频、音频和文件，同时顺手完成垃圾清理与相册整理的流量型工具 App。

**底层用户心智**

用户并不真正理解“数据恢复”的技术边界，他们购买的是一种情绪价值：
- 我删错了，能不能救回来
- 我手机很乱，能不能顺手清掉垃圾
- 我不懂技术，你替我一键扫描一下

这个产品正是把“后悔”和“侥幸”两个情绪点做成了下载动机。

## 4.2 目标用户

### 核心人群
- 误删照片/视频后的普通用户
- 手机存储告急，需要“恢复 + 清理”一体方案的人群
- 对文件系统和 Android 存储机制不了解的小白用户

### 次级人群
- 想恢复聊天媒体、缓存图片、旧截图的用户
- 低端机/老设备用户，尤其容易产生“是不是还在手机里”的恢复幻想

## 4.3 用户任务（JTBD）

- 当我误删重要照片时，我希望快速扫描并找回它，哪怕只是缓存版本
- 当我不确定文件是否真的丢了时，我希望有一个工具替我“全面搜一遍”
- 当我手机很乱时，我希望顺手清理重复图、截图和垃圾文件
- 当我很焦虑时，我希望工具给我一种“正在抢救数据”的确定感

## 4.4 功能模块拆解

### 模块1: 恢复入口
**目标**: 以高价值损失场景吸引首次打开
- 图片恢复
- 视频恢复
- 音频恢复
- 文件恢复

**推断交互**:
- 主页大卡片分类入口
- 点击后先授权，再开始扫描
- 扫描结果按类型列表化展示

### 模块2: 扫描结果页
**目标**: 让用户感知“找到了很多东西”
- 展示扫描数量
- 批量选择
- 恢复/导出/保存
- 可能带排序与预览

**关键问题**:
用户评论显示，部分结果可能只是“当前设备里本来就存在的文件”或历史不想要的文件，真正恢复价值感不高。

### 模块3: 文件管理
**目标**: 把一次性恢复行为延长成持续使用
- 已恢复文件统一列表
- 分类查看
- 再次整理
- 避免“恢复完又找不到”

### 模块4: 清理工具
**目标**: 从“救急工具”扩展成“高频工具”
- 垃圾清理
- 相似图片 / 截图清理
- 存储占用查看

这一步非常关键，因为真正的恢复需求低频，而清理需求高频，能支撑更多 DAU 和广告展示。

### 模块5: 通知与召回
**目标**: 提高回访与广告机会
- 评论显示存在高频通知
- 推断会用“发现更多可恢复文件”“存储空间告急”“清理建议”等文案做召回

## 5. 商业化与增长分析

## 5.1 商业化路径（事实 + 推断）

### 已确认事实
- Play 标记为 **Contains ads**
- 未看到明确订阅/内购主导证据

### 高可信推断
该产品主要靠以下模式变现：
1. 首页/结果页 Banner
2. 扫描完成后的插屏广告
3. 恢复/保存动作前激励或插屏
4. 高频通知带来的再打开
5. 清理类高频页面持续吃广告展示

## 5.2 增长逻辑

它的增长飞轮可能是：
- **SEO/ASO**: “recover photos”, “video recovery”, “deleted files” 等强需求词获量
- **高情绪点击率**: 用户处在“误删焦虑”状态，下载安装转化高
- **低技术门槛**: 一键扫描，立刻有反馈
- **清理场景留存**: 从恢复扩展到垃圾清理和相册整理
- **通知召回**: 持续触发复访

## 5.3 为什么评分只有 3.3

结合评论，低分原因大概率集中在：
- 承诺与结果不匹配，用户期待“真正恢复已删数据”，实际只是扫描现有文件/缓存
- 广告干扰重
- 通知打扰明显
- 用户对扫描结果缺乏解释，感觉被“虚假恢复”套路
- 网络差或低端机环境下，广告加载进一步损伤体验

## 6. 评论洞察

### 评论样本中看到的关键信号（事实）

1. 用户认为“找回的 200+ 照片”很可能只是过去删过但并不想要的旧照片，缺少价值解释
2. 用户明确抱怨广告过多，尤其在弱网环境里非常干扰
3. 用户抱怨通知频繁，而且似乎不授权通知就难以顺畅使用
4. 用户质疑所谓恢复结果只是手机里原本就有的文件，并非真正恢复

### 评论反映的核心矛盾

这个产品的真正矛盾不是“有没有扫描能力”，而是：
**用户买的是恢复奇迹，产品交付的是存储检索。**

这是需求包装与真实能力之间的裂缝。

## 7. 对我们做产品的启发

## 7.1 可以借鉴的

### 1. 用高情绪痛点切入
“误删找回”是非常强的下载触发点，远强于“文件管理”。

### 2. 把低频恢复需求接到高频清理需求
如果只做恢复，生命周期会很短；如果把恢复、清理、重复图片整理、存储管理串起来，产品会更有持续性。

### 3. 首页必须极其简单
这类用户焦虑且不懂技术，第一屏就应该是：
- 恢复照片
- 恢复视频
- 恢复音频
- 恢复文件

### 4. 扫描反馈要快
即便深度扫描需要时间，前几秒也必须给用户一个“发现了内容”的正反馈。

## 7.2 不建议模仿的

### 1. 夸大恢复能力
这是最伤信任的地方。尤其在文件恢复这个赛道，用户预期和技术真实能力差距很大，必须讲清边界。

### 2. 高强度广告打断
用户处在焦虑状态时，广告容忍度更低。强行插屏会显著拉低评分。

### 3. 用通知做强迫留存
短期 DAU 可能好看，长期口碑会塌。

### 4. 不解释“恢复来源”
如果结果里多数是缓存图、缩略图、已有文件，一定要明确标注来源，否则用户会觉得被骗。

## 8. 面向我方的 PRD

# PRD: 可信型手机照片/视频恢复助手

## 8.1 产品名称（暂定）
- 中文: 找回相册
- 英文: Restore Vault

## 8.2 产品愿景
帮助普通用户在误删照片/视频后，以**可理解、可信任、低打扰**的方式，最大化找回可恢复内容，并顺手完成手机媒体整理。

## 8.3 产品定位
区别于“泛恢复流量工具”，我们定位为：
**强调恢复结果解释、来源透明和低打扰体验的可信恢复助手。**

## 8.4 目标用户

### P0 用户
- 误删照片/视频后急于找回的 Android 普通用户

### P1 用户
- 手机相册混乱、截图很多、重复图多的用户
- 想找回聊天媒体或下载文件的用户

## 8.5 核心问题

现有竞品的主要问题：
- 说自己能恢复，但不解释恢复原理
- 找回结果价值低，混入大量已有文件
- 广告和通知严重打断
- 用户缺乏对“哪些能恢复、哪些不能”的心理预期

## 8.6 核心价值主张

1. **先说清楚，再扫描**
   - 告诉用户我们能找回什么，不能找回什么
2. **结果可解释**
   - 每个结果标注来源: 已有文件 / 缓存文件 / 缩略图 / 可恢复残留
3. **优先恢复真正有价值内容**
   - 优先显示近期删除、高清原图、用户最可能想找回的内容
4. **恢复后立即整理**
   - 一键加入相册、去重、归档

## 8.7 产品目标

### 北极星指标
- 成功找回并保存至少 1 个目标文件的用户占比

### 关键指标
- 扫描开始率
- 扫描完成率
- 恢复保存率
- 首次会话满意度
- 7日回访率（清理/整理场景）
- 差评率
- 广告触发后流失率

## 8.8 MVP 范围

### Must Have
1. 照片恢复
2. 视频恢复
3. 扫描结果来源标注
4. 文件预览
5. 批量恢复保存
6. 恢复后文件管理
7. 重复图/截图整理
8. 明确的权限说明页

### Nice to Have
1. 音频恢复
2. 下载文件恢复
3. 智能推荐最可能想找回的内容
4. 云备份提醒
5. 清理建议

### Not in MVP
1. iOS 版本
2. 桌面端
3. 深度 root 级恢复
4. 复杂会员体系

## 8.9 核心流程

### 流程A: 误删照片找回
1. 用户进入首页
2. 点击“找回照片”
3. 看到透明说明：将扫描相册、缓存、缩略图与可恢复媒体索引
4. 授权存储访问
5. 展示快速扫描结果
6. 结果按“高价值优先”排序
7. 用户预览并恢复
8. 恢复完成后进入整理页

### 流程B: 清理整理
1. 用户进入“整理空间”
2. 扫描重复图、截图、大文件
3. 批量选择删除/归档
4. 给出释放空间反馈

## 8.10 页面结构

### 首页
- 四个主 CTA: 找回照片 / 找回视频 / 找回音频 / 找回文件
- 辅助入口: 整理空间
- 价值承诺: 快速扫描、结果可解释、低打扰

### 扫描中页
- 扫描进度
- 已发现数量
- 提示不同来源说明
- 不插硬中断广告

### 结果页
- Tab: 高价值 / 全部 / 已有文件 / 缓存 / 缩略图
- 卡片字段: 缩略图、日期、大小、来源、恢复成功概率
- 操作: 预览 / 恢复 / 批量恢复

### 恢复完成页
- 保存到相册/文件夹
- 一键去重
- 查看相似项
- 提醒做备份

### 整理页
- 重复图片
- 截图
- 大视频
- 垃圾文件
- 已恢复文件

## 8.11 商业化策略

### 原则
- 不在用户最焦虑的瞬间硬插广告
- 先让用户感受到价值，再变现

### 建议方案
1. 免费版
   - 基础扫描 + 少量恢复
   - 结果页轻量 Banner
2. 看广告解锁
   - 单次批量恢复额外额度
3. Pro 版
   - 无广告
   - 批量恢复
   - 高级过滤与整理
   - 恢复记录归档

### 不建议
- 扫描开始即插屏
- 每次预览都触发广告
- 通知权限绑定核心功能

## 8.12 风险与边界

### 技术边界
- 非 root 情况下无法承诺真正恢复所有已彻底覆盖的数据
- Android 版本和厂商 ROM 差异会影响扫描结果

### 合规边界
- 必须清楚说明恢复机制
- 不得夸大“已删除必定恢复”
- 对相册、音视频、文件访问权限需充分告知用途

## 8.13 埋点建议

- 首页按钮点击
- 权限授权成功/失败
- 扫描开始/完成
- 扫描结果各来源数量
- 预览点击
- 恢复点击
- 恢复成功/失败
- 去重/清理动作
- 广告曝光后退出率
- 差评触发场景

## 8.14 首版优先级建议

### P0
- 照片/视频恢复主流程
- 来源解释
- 结果预览与批量恢复
- 基础整理能力

### P1
- 音频/文档恢复
- 智能排序
- 更好的恢复结果置信度评分

### P2
- 云备份联动
- 恢复前后对比报告
- 更细的文件来源解释模型
