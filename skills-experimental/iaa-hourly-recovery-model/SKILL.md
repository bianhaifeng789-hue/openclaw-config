# IAA 小时级回收模型

## 一、为什么你必须看小时级模型

你这种产品：
- 留存差
- 首日价值高
- 后续尾值薄

所以不能只看 D0。
你真正该看的是：

```text
H1 → H3 → H6 → H12 → D0
```

因为这能提前判断：
1. 这个量是不是对的
2. 这个创意是不是带来了“首小时高价值用户”
3. 这波量是否值得继续放
4. 是否应该 6 小时内直接砍掉

---

## 二、完整小时级回收公式

### 累计ROI

```text
H1 ROI = H1 Revenue / Spend
H3 ROI = (H1 + H2-H3 Revenue) / Spend
H6 ROI = (H1 + H2-H6 Revenue) / Spend
H12 ROI = (H1 + H2-H12 Revenue) / Spend
D0 ROI = Day0 Revenue / Spend
```

### 增量价值拆解

```text
D0 Revenue = H1 + H2-H3 + H4-H6 + H7-H12 + H13-H24
```

对你这类低留存IAA产品，常见分布：

```text
H1: 20% ~ 35% 的 D0价值
H3: 35% ~ 50% 的 D0价值
H6: 50% ~ 70% 的 D0价值
H12: 70% ~ 85% 的 D0价值
H24: 100% 的 D0价值
```

如果是超高意图工具类（文件修复/清理），首小时占比可能更高：

```text
H1: 25% ~ 40%
H3: 45% ~ 60%
H6: 60% ~ 75%
H12: 80% ~ 90%
```

## 三、你这个场景的小时级模板

### 1）文件修复（D0回收 85%）

推荐基准模型：

```text
D0 ROI = 85%

拆解：
H1 = 22%
H3 = 40%
H6 = 58%
H12 = 73%
H24 = 85%
```

也就是说，如果一条量是健康的，通常会长成：

| 时间 | 累计ROI |
|------|--------|
| H1 | 20%-25% |
| H3 | 35%-45% |
| H6 | 50%-60% |
| H12 | 68%-75% |
| D0 | 80%-85%+ |

#### 判断线

```text
H1 < 18%   → 量偏差，危险
H3 < 35%   → 大概率D0起不来
H6 < 50%   → 基本无法到85%
H12 < 68%  → D0很难过80%
```

---

### 2）清理（D0回收 65%）

推荐基准模型：

```text
D0 ROI = 65%

拆解：
H1 = 15%
H3 = 28%
H6 = 42%
H12 = 55%
H24 = 65%
```

对应判断线：

| 时间 | 累计ROI |
|------|--------|
| H1 | 12%-18% |
| H3 | 22%-30% |
| H6 | 35%-45% |
| H12 | 48%-58% |
| D0 | 60%-65% |

#### 判断线

```text
H1 < 12%   → 很差，优先停
H3 < 22%   → 基本没希望
H6 < 35%   → 大概率D0起不来
H12 < 48%  → D0大概率低于60%
```

## 四、小时级反推模型

如果你知道目标 D0 ROI，可以反推小时阈值。

### 通用公式

```text
H1 Target = D0 Target × H1占比
H3 Target = D0 Target × H3占比
H6 Target = D0 Target × H6占比
H12 Target = D0 Target × H12占比
```

### 文件修复（目标 D0=85%）

按中位占比：

```text
H1 = 85% × 26% = 22.1%
H3 = 85% × 47% = 39.95%
H6 = 85% × 68% = 57.8%
H12 = 85% × 86% = 73.1%
```

### 清理（目标 D0=65%）

按中位占比：

```text
H1 = 65% × 23% = 15.0%
H3 = 65% × 43% = 28.0%
H6 = 65% × 65% = 42.3%
H12 = 65% × 85% = 55.3%
```

## 五、小时级投放决策规则

### 文件修复

```text
H1 ≥ 22%  → 保留观察
H3 ≥ 40%  → 可以继续跑
H6 ≥ 58%  → 可准备加量
H12 ≥ 73% → D0大概率健康
```

#### 停量规则

```text
H1 < 18%  → 直接标红
H3 < 35%  → 大概率停
H6 < 50%  → 建议停
```

### 清理

```text
H1 ≥ 15%  → 可观察
H3 ≥ 28%  → 继续跑
H6 ≥ 42%  → 勉强合格
H12 ≥ 55% → D0可到65%
```

#### 停量规则

```text
H1 < 12%  → 直接停
H3 < 22%  → 停
H6 < 35%  → 停
```

## 六、小时级LTV建模

### 核心思路

你可以把总LTV先拆成：

```text
LTV = H1 + H2-H3 + H4-H6 + H7-H12 + H13-H24 + D1+tail
```

对低留存产品：

```text
D0 ≈ 总LTV的 65% ~ 85%
```

所以：

```text
总LTV = D0 / D0占总LTV比重
```

再结合小时数据，可以更早预测：

```text
预测D0 = 当前小时收入 / 当前小时进度占比
预测总LTV = 预测D0 / D0占总LTV比重
```

### 举例：文件修复

如果：

```text
H6 ROI = 58%
且 H6 通常占 D0 的 68%
```

则：

```text
预测 D0 ROI = 58% / 0.68 = 85.3%
```

如果 D0 占总LTV 80%，则：

```text
预测总ROI = 85.3% / 0.8 = 106.6%
```

说明这波量可投。


### 清理

```text
目标线：
H1 15%
H3 28%
H6 42%
H12 55%
D0 65%

策略：
- H1 不达标直接停
- H3 不达标直接停
- 不做长观察
```

## 九、一句话总结

```text
你这类产品，小时级回收比日级回收更重要。

文件修复看 H6 能不能到 58%。
清理看 H3 能不能到 28%。
到不了，就别等D0，直接处理。
```
