# Harness Deploy Quick Checklist

适合开工前、提测前、投放前快速过一遍。

## 0. 任务类型
- [ ] 修复型
- [ ] 迁移型
- [ ] 新增型
- [ ] 验收型

## 1. 先问三个问题
- [ ] 现在已有啥
- [ ] 还缺啥
- [ ] 哪个缺口最影响投放

## 2. P0 主链路
- [ ] 入口在
- [ ] 调度在
- [ ] 执行在
- [ ] 返回在
- [ ] 错误收口在

## 3. P1 防炸件
- [ ] 长输出可控
- [ ] 大结果可落盘
- [ ] 缺参能兜底
- [ ] 参数错能自动修正
- [ ] 子任务/工具调用不会直接失控

## 4. 真实投放判断
- [ ] 不是“文件存在但没接入口”
- [ ] 不是“能跑 happy path 但边界全炸”
- [ ] 不是“看起来覆盖了，其实没真实调用”

## 5. 最小验收
- [ ] 正常短任务跑通
- [ ] 长输出任务跑通
- [ ] 异常/边界任务可控

## 6. 可以投放的标准
- [ ] 可调用
- [ ] 可执行
- [ ] 可返回
- [ ] 可兜底

## 一句话
**先补骨架，再补防炸件，再看增强项。**
