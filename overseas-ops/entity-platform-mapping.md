# 主体-平台映射表（第一版）

> 用途：回答“哪个主体适合哪个平台”“某平台还缺什么资料”“某产品当前可以挂到哪个主体”这类问题。

## 一、映射字段

| 字段 | 说明 |
|---|---|
| 映射ID | 唯一编号 |
| 平台名称 | 目标平台 |
| 平台类型 | 广告平台 / 中介平台 / 商店 / 商务平台 |
| 产品/业务线 | 对应产品 |
| 推荐主体ID | 推荐使用的主体 |
| 主体国家/地区 | 主体注册地 |
| 是否满足地区要求 | 是 / 否 / 待确认 |
| 是否满足资料要求 | 是 / 否 / 部分满足 |
| 当前缺失项 | 还差哪些资料 |
| 风险等级 | 低 / 中 / 高 |
| 备注 | 驳回经验 / 限制 / 特殊要求 |
| 状态 | 待评估 / 可提交 / 已提交 / 被驳回 |
| 最近更新 | 更新时间 |

## 三、使用规则

1. 先确认平台地区要求，再选主体
2. 同一个平台允许保留多个候选主体，但必须标主推荐主体
3. 缺失项必须明确到文件或字段级
4. 驳回过的平台要保留历史经验，别重复踩坑
