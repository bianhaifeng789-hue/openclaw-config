- 用户明确要求检查 `~/Desktop/oc`（另一台电脑的 OpenClaw 配置）并尽可能借鉴到当前机器
- 判断结果：最值得迁移的是 `skills/` 与 `impl/dist/`，不建议直接覆盖当前机器已有 `scripts/`，因为当前机器已有较多自研心跳、自愈、同步、锁管理脚本
- 迁移策略确定为：**只新增、不覆盖**，优先补齐能力资产，避免破坏当前可运行逻辑

### 配置差异分析结论
- 桌面机与当前机之间，真正影响功能启用的 `.openclaw` 级配置差异很小

### 规则合并决策
- 判断 `HEARTBEAT.md` 当前版本已是较优合并版，不应粗暴回退
- 判断 `MEMORY.md` 应保留当前机器真实角色、任务状态与长期记忆，不应覆盖
- 决定将后续规则增强聚焦在：
  - `AGENTS.md`：补强 heartbeat 执行顺序 / 返回规则 / 后台维护优先级
  - `HEARTBEAT.md`：补充 system reminder 的内部处理 / 外发规则

### System Reminder 处理经验沉淀
- 对于 `session_lock_cleanup`、`process_health_monitor`、`self_healing_check` 这类提醒：默认应视为后台例行维护
- 如果系统要求“内部处理且不要转述用户”，应安静处理，正常时返回 `HEARTBEAT_OK`
- 如果系统要求“Please relay this reminder to the user”，则可以用自然语言简洁转述
- 对于 `proactive_compact_check`、`context_pressure_check` 这类上下文压力提醒：优先内部检查，不默认外发；只有在存在风险、异常、修复动作或用户感知价值时才主动汇报
- 多个 reminder 同轮触发时，应尽量合并处理，避免刷屏

### 用户偏好 / 当前意图
- 用户同意继续做“迁移后接线体检”和“规则合并”
- 用户明确授权继续执行规则合并，并要求“做完告诉我结果”

- 2026-04-14 15:24 CST：已定位后台刷屏提醒的更深层根因：maintenance 类 cron/job 虽多为 `lastDeliveryStatus: not-requested`，但因为使用 `sessionTarget: "main" + wakeMode: "now"`，仍会把 system event 直接注入主会话时间线，造成频繁系统提醒。

### compact-cli.js 创建
- **位置**: impl/bin/compact-cli.js
- **功能**: 统一压缩管理器，借鉴 Claude Code 多层压缩

**Claude Code 借鉴的关键常量：**
| 常量 | 值 | 用途 |
|------|-----|------|
| AUTOCOMPACT_BUFFER_TOKENS | 13,000 | 压缩触发缓冲 |
| MAX_OUTPUT_TOKENS_FOR_SUMMARY | 20,000 | 摘要最大输出 |
| WARNING_THRESHOLD_BUFFER | 20,000 | 预警告缓冲 |
| ERROR_THRESHOLD_BUFFER | 20,000 | 紧急压缩缓冲 |
| MAX_RETRY_ATTEMPTS | 3 | 重试次数 |
| COMPACT_TIMEOUT_MS | 30,000 | 超时保护 |

**多层压缩策略：**
| Level | 名称 | 触发条件 | 动作 |
|-------|------|----------|------|
| 0 | TimeBasedMC | 距上次 assistant > 60min | 清理旧 tool results |
| 1 | MicroCompact | 超过 auto 阈值 | 选择性清理 |
| 2 | SessionMemoryCompact | 超过 warning 阈值 | MEMORY.md 压缩 |
| 3 | SummaryCompact | 超过 error 阈值 | 生成摘要（最后手段） |

### HEARTBEAT.md 更新
新增 3 个压缩任务：
- `context-pressure-check` (30m, high) - 检查上下文压力
- `time-based-mc` (1h, medium) - 时间触发微压缩
- `memory-compact` (24h, low) - 增强，使用 compact-cli.js

### 阈值配置（glm-5）
- **contextWindow**: 200,000
- **effectiveWindow**: 180,000 (减去摘要输出)
- **autoThreshold**: 167,000 (93%) - 自动触发
- **warningThreshold**: 160,000 (89%) - 预警告
- **errorThreshold**: 160,000 (89%) - 紧急压缩

### 统计更新
- 心跳任务：21 → **24**
- impl/bin 脚本：48 → **49**（新增 compact-cli.js）

### 关键决策
- OpenClaw 内置压缩机制不自动调用我们部署的脚本
- 需要通过心跳系统或 cron 定期调用 compact-cli.js
- 借鉴 Claude Code 的渐进式压缩策略，避免一次性全量压缩
