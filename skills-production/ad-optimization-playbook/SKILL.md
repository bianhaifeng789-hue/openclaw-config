# 广告调优配置SOP

## 一、瀑布流 vs Bidding

### 瀑布流（Waterfall）
```
原理：按eCPM从高到低依次请求广告网络

优点：
✅ 控制精确
✅ 可以手动调整优先级
✅ 适合稳定收益

缺点：
❌ 请求慢（需依次请求）
❌ 不透明（无法看到真实竞价）
❌ 需要频繁手动调整

配置步骤：
1. 进入中介平台（AdMob Mediation / LevelPlay）
2. 创建中介组
3. 添加广告网络
4. 设置eCPM顺序（从高到低）
5. 设置延迟时间（默认0秒）
```

### Bidding（竞价）
```
原理：所有广告网络同时竞价，最高价获胜

优点：
✅ 请求快（一次请求）
✅ 透明（看到真实竞价价格）
✅ 自动优化（无需手动调整）
✅ 收益更高（真实竞价）

缺点：
❌ 需要广告网络支持竞价
❌ 新号需要积累展示量才能进入竞价池

配置步骤：
1. 进入中介平台（MAX / LevelPlay）
2. 启用Bidding
3. 添加支持竞价的网络
4. 设置竞价权重（或自动）
5. 发布中介组
```

### 推荐策略
```
新号：先用瀑布流，积累展示量
成熟账号：切换到Bidding，收益更高

Bidding优先顺序：
MAX > LevelPlay > AdMob Mediation
```

## 二、eCPM Floor设置

### 什么是eCPM Floor
```
eCPM Floor = 广告展示的最低价格要求

作用：
- 防止低价广告展示
- 提高整体收益
- 控制广告质量

风险：
- Floor过高 → 展示量下降
- Floor过低 → 收益下降
```

### 设置方法
```
Step 1: 观察历史eCPM
- 查看过去7天的eCPM数据
- 找到平均值和峰值

Step 2: 设置Floor
- 初期：设置为平均eCPM的70%
- 例如：平均$8 → Floor $5.6

Step 3: 测试调整
- 每3天调整一次
- 观察展示量和收益变化
- 找到最佳平衡点
```

### 各广告类型Floor建议（美国）
```
激励视频：$8-12
插屏：$5-8
Banner：$1-2

新号建议：
激励视频：$5
插屏：$3
Banner：$0.5
```

## 三、分国家策略

### 为什么分国家
```
不同国家eCPM差异大：
- 美国：$10-15
- 英国：$8-12
- 德国：$6-10
- 印度：$0.5-1
- 东南亚：$0.2-0.5

分国家策略：
- 高价值国家：高Floor + 更多广告位
- 低价值国家：低Floor + 控制频控
```

### 配置方法
```
Step 1: 创建国家分组
- Tier 1：美国、英国、加拿大、澳大利亚
- Tier 2：德国、法国、日本、韩国
- Tier 3：其他发达国家
- Tier 4：发展中国家

Step 2: 设置不同Floor
- Tier 1：激励视频 $10，插屏 $7
- Tier 2：激励视频 $8，插屏 $5
- Tier 3：激励视频 $5，插屏 $3
- Tier 4：激励视频 $1，插屏 $0.5

Step 3: 创建中介组
- 在中介平台创建多个中介组
- 每个中介组对应一个国家分组
- 设置不同的广告网络和Floor
```

## 四、广告位策略

### 激励视频
```
最佳时机：
- 关卡失败后（复活）
- 关卡通关后（奖励）
- 商店购买道具（替代付费）
- 每日签到奖励

频控建议：
- 每小时≤3次
- 每次间隔≥20秒
- 不要强制展示

收益优化：
- 设置高Floor（$8-12）
- 优先竞价网络
- 添加多网络竞争
```

### 插屏
```
最佳时机：
- 关卡切换时
- 游戏暂停时
- 功能页面退出时

频控建议：
- 每小时≤2次
- 每次间隔≥60秒
- 不要在游戏进行中展示

收益优化：
- Floor中等（$5-8）
- 瀑布流 + 竞价混合
- 控制展示时机
```

### Banner
```
位置建议：
- 底部（最佳）
- 顶部（次选）
- 不要遮挡游戏内容

频控建议：
- 持续展示
- 自动刷新（60秒）
- 不要手动关闭

收益优化：
- Floor低（$1-2）
- 添加多网络
- 控制位置不遮挡
```

## 五、频控设置

### 什么是频控
```
频控 = 控制广告展示频率

作用：
- 防止用户体验下降
- 防止广告疲劳
- 提高单次展示价值

常见频控：
- 每小时展示次数限制
- 两次展示间隔时间
- 每日展示次数限制
```

### 频控建议（不同广告类型）
```
激励视频：
- 每小时：≤3次
- 间隔：≥20秒
- 每日：≤10次

插屏：
- 每小时：≤2次
- 间隔：≥60秒
- 每日：≤6次

Banner：
- 持续展示
- 刷新：60秒
- 不设上限
```

## 六、首曝时机

### 什么是首曝
```
首曝 = 新用户首次看到广告的时机

作用：
- 影响留存率
- 影响用户体验
- 影响早期收益

策略：
- 新用户不要立即展示广告
- 等用户熟悉应用后再展示
- 首曝时机影响长期收益
```

### 首曝时机建议
```
超休闲游戏：
- 首曝：第2-3关卡（约30秒后）
- 留存影响最小

休闲游戏：
- 首曝：第5关卡后（约2分钟）
- 用户体验更好

中重度游戏：
- 首曝：首次关卡失败后（约5分钟）
- 不影响沉浸感

工具类APP：
- 首曝：完成核心功能后（约1分钟）
- 不干扰使用
```

## 七、收益异常排查

### 收益突然下降
```
排查步骤：

1. 检查展示量
   - 展示量是否下降？
   - 原因：应用问题、SDK问题、网络问题

2. 检查eCPM
   - eCPM是否下降？
   - 原因：竞价问题、Floor问题、季节因素

3. 检查广告网络
   - 各网络是否正常响应？
   - 某个网络可能异常

4. 检查配置
   - Floor是否设置过高？
   - 频控是否设置过严？
   - 中介配置是否正确？

5. 检查季节因素
   - 周末/节假日eCPM下降？
   - 季度末预算减少？
```

### 展示量突然下降
```
排查步骤：

1. 检查应用
   - 应用是否正常运行？
   - 用户是否正常使用？

2. 检查SDK
   - SDK是否正常初始化？
   - 广告是否正常加载？

3. 检查网络
   - 广告网络是否响应？
   - 网络连接是否正常？

4. 检查频控
   - 频控是否设置过严？
   - 用户是否触发广告条件？
```

## 八、A/B测试方法

### 测试步骤
```
Step 1: 创建对照组
- A组：当前配置
- B组：新配置

Step 2: 分配流量
- 50%流量给A组
- 50%流量给B组

Step 3: 运行测试
- 测试周期：7-14天
- 收集数据

Step 4: 分析结果
- 比较收益、eCPM、展示量
- 选择最佳配置
```

### 测试变量
```
可测试变量：
- eCPM Floor
- 频控设置
- 广告位位置
- 首曝时机
- 中介网络组合
- 瀑布流顺序
```
