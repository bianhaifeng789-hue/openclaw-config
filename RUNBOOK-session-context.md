# RUNBOOK-session-context.md

OpenClaw 的 session、上下文压缩、长线程收口规则。

## 目标

让主会话保持轻量，把排障、实验、长日志与后台检查从主线程剥离。

---

## 1. 会话分流规则

### 主会话只保留
- 结论
- 决策
- 下一步
- 用户需要知道的变化

### 独立线程 / 子会话处理
- 长日志
- 多轮试错
- 配置深挖
- bug 定位
- 任何连续 3 步以上的技术排查

### heartbeat / cron 只处理
- check
- route
- notify

不处理：
- 长排障
- 复杂修复
- 长链路验证

---

## 2. 上下文压缩规则

### 默认原则
- 长输出不要长期留在主线程
- 过程写文件，主线程只留摘要
- 一段工作结束后必须收口

### 收口模板
1. 现象
2. 判断
3. 改了什么
4. 验证结果
5. 剩余尾巴
6. 下一步（若有）

### 适合外置到文件的内容
- 完整命令输出
- config diff
- 多轮排障时间线
- 大段安全审计结果

---

## 3. 何时必须分流

满足任一条件就分流：
- 需要连续执行 3 步以上命令
- 需要同时看 status / doctor / logs / config
- 需要多次编辑配置并验证
- 需要读大量文件/代码
- 已经开始讨论实现细节而不是用户决策

---

## 4. 何时可以直接收线

满足以下条件可以收线：
- 主风险项已解决
- 剩余项是展示层噪音或非阻塞提醒
- 结论已写入 runbook / memory
- 后续处理不影响当前正常使用

---

## 5. 与 memory 的关系

- daily notes 记过程
- MEMORY.md 记长期规律
- runbook 记可复用方法
- HEARTBEAT.md 记自动化边界

不要把这四类内容混在一起。
