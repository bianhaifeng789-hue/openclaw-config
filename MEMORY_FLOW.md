# MEMORY_FLOW.md - 记忆流转规则

目标：明确什么信息该写到哪里，减少主会话长期背负历史。

## 1. 写入 `memory/YYYY-MM-DD.md`
适用于：
- 当天发生的事件
- 当前排障过程中的关键进展
- 一次修复的执行记录
- 需要当天回看的上下文

规则：
- 优先写摘要，再写细节
- 重大事件优先按 `INCIDENT_TEMPLATE.md` 组织

## 2. 提炼进 `MEMORY.md`
适用于：
- 重复出现的规律
- 长期应该避免的坑
- 已验证有效的长期策略
- 对未来多次有帮助的决策与经验

规则：
- 不抄流水账，只写长期有价值的结论
- 一条经验如果未来三次以上可能用到，就值得进入 `MEMORY.md`

## 3. 写入 `RUNBOOK.md`
适用于：
- 固定排障顺序
- 稳定可复用的检查命令
- 面向“下次怎么做”的手册型内容

规则：
- 保持最短可执行
- 避免把 runbook 写成长篇事故复盘

## 4. 写入 `SCRIPTS.md`
适用于：
- 本地自动化脚本的用途、风险、状态
- 哪些脚本已废弃 / observe-only / 禁止恢复

## 5. 写入 `INSIGHTS.md`
适用于：
- 用户反馈与高频抱怨
- 竞品变化
- 市场观察
- 对产品有意义的外部信号与洞察

规则：
- 优先记录对决策有帮助的信号
- 洞察不直接等于长期记忆；只有形成重复规律后再考虑提炼进 `MEMORY.md`

## 6. 写入 `DECISIONS.md`
适用于：
- 重要产品取舍
- 功能优先级判断
- onboarding / paywall / 文案 / roadmap 等重要决定

规则：
- 重点记录“为什么这样选”
- 不要只把最终结论留在聊天里

## 7. 写入 `EXPERIMENTS.md`
适用于：
- 增长实验
- onboarding / pricing / paywall / 文案 / listing 等产品实验

规则：
- 记录假设、目标指标、结果和结论
- 实验结果默认先留在 `EXPERIMENTS.md`，重复出现的规律再提炼进 `MEMORY.md`

## 8. 何时收口当前线程
满足以下任一条件时，优先考虑收口并外化结论：
- 同一线程里已经混入多个主题（例如配置、日志、脚本、安全、计划）
- 原始日志/工具输出明显偏长
- 已经完成一轮诊断 + 修复 + 验证
- 已出现 context overflow / compaction 压力

默认动作：
1. 写简短摘要
2. 更新相应文件
3. 如有需要，换新线程继续下一个主题
