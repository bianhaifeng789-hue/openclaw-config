# 防止重复消息规则

## 问题现象

每次回复发2条相似内容，用户困惑。

## 根本原因

1. **习惯性分步报告** - 先发进度卡片，再发结果卡片
2. **工具失败提示** - 发失败提示，重试成功又发成功提示
3. **心跳重复报告** - 检查结果 + 任务状态分开发

## 解决方案

**核心原则**: **一张卡片原则**

### 规则表

| 场景 | ❌ 旧模式（重复） | ✅ 新模式（单卡片） |
|------|-----------------|-------------------|
| 完成任务 | 先发进度卡片<br/>再发结果卡片 | 合并为一张结果卡片 |
| 工具失败 | 发失败提示<br/>重试后发成功提示 | 静默重试，只发最终结果 |
| 心跳检查 | 发检查结果<br/>发任务状态 | 合并为一张状态卡片 |

### 实施要点

**长任务（>5分钟）**:
- 开始 → 发1张（预估时间）
- 进度 → 每5分钟发1张（不重复）
- 完成 → 发1张（结果）
- **总计**: 3张卡片，不重复

**短任务（<5分钟）**:
- 开始 → 静默执行
- 完成 → 发1张（结果）
- **总计**: 1张卡片

**工具失败**:
- 失败 → 静默重试（不发失败提示）
- 成功 → 发1张（结果）
- **总计**: 1张卡片

**心跳检查**:
- 正常 → HEARTBEAT_OK（静默）
- 发现 → 发1张（发现内容）
- **总计**: 0-1张卡片

## 典型错误示例

### ❌ 错误：重复发送

```
我: 🔍 开始任务（预估10分钟）
我: ✅ 进度25% (1/4) 分析完成    ← 重复！
我: ✅ 进度50% (2/4) 提取完成    ← 重复！
我: ✅ 进度75% (3/4) 创建完成    ← 重复！
我: ✅ 完成（100%）结果：14 Skills
（共 5 张卡片，信息冗余）
```

### ✅ 正确：精简发送

```
我: 🔍 开始任务（预估10分钟）
    我会定期报告进度。

[5分钟后心跳自动发送进度卡片]

我: ✅ 完成（100%）结果：14 Skills
（共 3 张卡片，信息清晰）
```
