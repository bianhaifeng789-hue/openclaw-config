## Phase 33: Context Management 系统实施 (2026-04-13 21:36-21:53)

### 背景

基于 Claude Code 源码分析（docs/Claude-Code-Analysis.md），实施上下文防溢出系统。

### 新创建文件

| 文件 | 路径 | 功能 |
|------|------|------|
| **timeBasedMC.js** | ~/.openclaw/skills/compact/scripts/ | 时间触发微压缩（30min idle 清理） |
| **autoCompactTrigger.js** | ~/.openclaw/skills/auto-compact/scripts/ | 多层自动压缩触发机制 |
| **compactHooks.js** | ~/.openclaw/skills/hook-system/scripts/ | Pre/Post Compact 生命周期 Hooks |
| **contextManager.js** | ~/.openclaw/skills/context-management-integration/scripts/ | 统一上下文管理器 |
| **SKILL.md** | ~/.openclaw/skills/context-management-integration/ | 集成 Skill 文档 |
| **context-management-config.json** | ~/.openclaw/ | 全局配置文件 |

### 压缩策略层级

```
Level 0: TimeBasedMC (30min idle) → 清理旧工具结果
Level 1: MicroCompact → 清除大工具结果 (>10K)
Level 2: SelectiveCompact → 去除重复
Level 3: SummaryCompact → 生成摘要 (最后手段)
```

### 阈值配置（GLM-5）

| 阈值 | 值 | 说明 |
|------|-----|------|
| Context Window | 128K | 总上下文窗口 |
| Auto Threshold | 115K | 自动压缩触发 |
| Warning Threshold | 96K | 提前警告 |
| Error Threshold | 115.2K | 紧急压缩 |

### Hooks 功能

- **PreCompact**: 保存状态、自定义指令、预处理消息
- **PostCompact**: 恢复状态、日志统计、发送通知

### API 示例

```javascript
const { ContextManager, quickShouldCompact, quickStatus } = 
  require('~/.openclaw/skills/context-management-integration/scripts/contextManager.js');

// 检查状态
const status = quickStatus(currentTokens, 'glm-5');

// 自动管理
const manager = new ContextManager({ defaultModel: 'glm-5' });
manager.setMessages(messages);
const result = await manager.checkAndCompact();
```

### 防溢出策略

1. **预防**: TimeBasedMC 空闲时自动清理
2. **早期检测**: Warning threshold (75%) 提醒
3. **自动响应**: Auto threshold (85%) 自动压缩
4. **紧急处理**: Error threshold (90%) 立即强制

### 效果预估

| 策略 | 平均节省 | 成本 |
|------|---------|------|
| TimeBasedMC | 5-10K | 极低 |
| MicroCompact | 10-30K | 低 |
| SelectiveCompact | 5-15K | 低 |
| SummaryCompact | 50-100K | 高 |
