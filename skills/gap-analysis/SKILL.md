# Gap 分析技能

当用户提到 **Gap 分析、对标差距、竞品对比、别人有我们没有、该学什么、不学什么、版本优先级** 时，使用本技能。

## 1. 适用场景
- 产品对标
- 版本规划
- MVP 范围取舍
- 竞品启发整理
- 跟进 / 不跟进决策

## 2. 常见触发句
- “做个 gap 分析”
- “别人有啥我们没有”
- “哪些值得跟，哪些别跟”
- “帮我排优先级”
- “这个功能我们要不要做”

## 3. 输入
可基于以下材料开展：
- PRD
- 竞品逆向分析
- 竞品分析报告
- 页面截图 / 使用体验
- 业务目标 / 数据指标

## 4. 输入不足时
如果没有明确对标对象或版本目标，先补齐：
- 对标对象
- 分析目标
- 比较维度
- 目标版本

若只能拿到部分材料，应先做“初步 Gap 分析”，标明待确认项。

## 5. 分析目标
目标不是列差异，而是产出决策依据：
- 哪些差距最影响用户价值 / 转化 / 留存 / 收入
- 哪些应该立刻补齐
- 哪些值得观察但不急着做
- 哪些不适合照搬

## 6. 标准分析维度
至少覆盖：
- 功能 Gap
- 体验 Gap
- 商业化 Gap
- 增长 Gap
- 数据 Gap
- 技术 Gap

## 7. 推荐输出结构
```markdown
# Gap 分析
## 1. 结论摘要
## 2. 对标对象说明
## 3. 总体差距盘点
## 4. 功能 Gap
## 5. 体验 Gap
## 6. 商业化 Gap
## 7. 增长 Gap
## 8. 数据 Gap
## 9. 技术 Gap
## 10. 学什么 / 不学什么
## 11. Now / Next / Later / Never
## 12. 版本建议
```

## 8. 最低输出要求
至少必须给出：
- 3 个关键 Gap
- 每个 Gap 的影响说明
- 跟 / 不跟结论
- Now / Next 两级优先级建议

## 9. 输出原则
- 不要只说“竞品有，我们没有”
- 要解释差距影响什么
- 要给优先级与建议动作
- 要区分“值得跟进”和“没必要跟”
- 最终能进入版本规划

## 10. 联动技能
- 竞品输入不足时，先结合 `competitive-analysis`
- 逆向数据由上游 `google-play-to-prd` 提供，本 skill 只消费逆向结果，不直接调用逆向工具
- 需要落到方案时，输出传给 `prd-generator`
