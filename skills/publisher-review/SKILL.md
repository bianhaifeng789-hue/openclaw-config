# Publisher审核指南

APP/网站接入广告变现平台的审核标准。

## 审核流程

```
1. 接收申请 → 查看基本信息
2. 检查APP合规性 → 内容 + Store页面
3. 检查SDK集成 → 技术验证
4. 评估流量质量 → 日活 + 地区分布
5. 内容审核 → 无违规内容
6. 最终决策 → 批准/拒绝/补充材料
```

---

## APP审核标准

### 基本信息

| 项目 | 检查内容 |
|------|----------|
| **APP名称** | 无违规关键词 |
| **Bundle ID** | 正确格式 |
| **Store链接** | 已上架 |
| **开发者** | 真实信息 |

### 内容合规

```
✅ 允许内容：
- 游戏、娱乐、工具
- 新闻、教育、购物
- 社交（合规）

❌ 禁止内容：
- 仇恨言论
- 暴力内容
- 成人内容
- 侵权内容
- 仿冒品牌
```

### Store页面检查

| 检查项 | 要求 |
|--------|------|
| **截图** | 真实截图，无误导 |
| **描述** | 与APP功能一致 |
| **分类** | 正确分类 |
| **评分** | 3.0+（建议） |
| **评论** | 无大量负面评论 |

## SDK集成检查

### 集成验证

```
检查项：
✅ SDK版本（最新稳定版）
✅ 初始化代码正确
✅ 广告位ID正确
✅ 测试模式关闭
✅ 无内存泄漏
✅ 无崩溃问题
```

### 常见问题

```
❌ 问题：广告不显示
原因：SDK未初始化、广告位ID错误
解决：检查初始化代码

❌ 问题：点击率异常高
原因：误触设计、虚假点击
解决：检查广告位置设计

❌ 问题：收益异常低
原因：流量质量差、地区不符
解决：分析流量来源
```
